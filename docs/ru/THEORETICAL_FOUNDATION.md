## 1. Математика TAT

### 1.1 Основные константы

| Константа | Значение | Где проверяется |
|-----------|----------|-----------------|
| Фазовый угол θ | 1.987 | `TAT_CORE.json` → `constants.phase_angle` |
| Нижняя температура T_min | 0.3 | `TAT_CORE.json` → `constants.coherence_threshold` (нижняя граница) |
| Верхняя температура T_max | 0.7 | `TAT_CORE.json` → `constants.temperature` |
| Множитель веса связей | 4 | `TAT_CORE.json` → `dialog_lifecycle.connection_weight_multiplier` |
| Масштаб мнимой части | 0.3 | `TAT CORE v2.2` → `BASE_IMAG_SCALE` |
| Контролируемый шум | 0.7% | Выведено: T_max / 100, соответствует порогам divergence trace |

### 1.2 Ключевые формулы

**Комплексный вес:**
```
ψ_k = w_k · e^(i·θ_k)
```
где θ_k = θ + k · 0.1 (шаг фазы головы).

Проверяется в: `TATCore.py` → `complex_weight()`

**Когерентность:**
```
coherence = |w1 · conj(w2)| / (|w1| · |w2| + ε)
```
Проверяется в: `TATCore.py` → `coherence()`

**Divergence trace:**
```
divergence = Position − Coherence
```
Проверяется в: `TAT-ROOT/data/tat7_divergence_trace.csv`

**Операционная оболочка:**
```
Нижняя граница: 93% = 100% − 7% (7 голов, T_max = 0.7, шум 0.7%)
Верхняя граница: 96% = 100% − 4% (connection_weight_multiplier = 4)
```
Проверяется в: `TAT-ROOT/docs/ru/KETTLE_PRINCIPLE.md`

### 1.3 Пороги эскалации

| Дивергенция | Голов | Решение Gate |
|------------|-------|---------------|
| < 0.10 | 3 | Consolidate |
| 0.10 – 0.30 | 5 | Escalated (защитная коррекция) |
| > 0.30 | 7 | Withhold (если harmony молчит) |

Проверяется в: `TAT-7 Comparative Benchmark` → `TAT-ROOT/data/`

## 2. Физика TAT

### 2.1 Первый закон термодинамики в TAT

```
ΔU = Q − A
```
- ΔU: Изменение когерентности системы.
- Q: Тепло — энтропия градиентов, k_B · T.
- A: Работа — активация голов, фазовый переход, resonance replay.

Проверяется в: `TAT CORE v2.2` → `predict_pair()` — когерентность вычисляется как интерференция комплексных векторов.

### 2.2 Фундаментальные физические константы

| Константа | Символ | Значение | Роль в TAT |
|-----------|--------|----------|-------------|
| Постоянная Планка | h | 6.626 × 10⁻³⁴ Дж·с | Квант информации, минимальное действие |
| Постоянная Больцмана | k_B | 1.381 × 10⁻²³ Дж/К | Тепловая флуктуация, цена шума |
| Постоянная Эйлера-Маскерони | γ | 0.57721 | Поправка в масштабировании энтропии |

Проверяется в: `TAT_CORE.json` → `constants`

### 2.3 Вывод фазового угла

```
θ = arctan2(h − k_B, h + k_B)
```

Геометрически: θ ≈ 73/37 ≈ 1.97297 + поправка на шум 0.7%.

Проверяется в: `TAT_CORE.json` → `formulas.triple_theta`

## 3. Термодинамика TAT

### 3.1 Температурная шкала и фазовые переходы

| Режим | Температура | Голов | Поведение |
|--------|-------------|-------|-----------|
| Холодный (твёрдый) | T < 0.3 | 3 | Память заморожена, минимальное обучение |
| Рабочий (жидкий) | 0.3 ≤ T < 0.7 | 5 | Пластичность и память в равновесии |
| Горячий (газ) | T ≥ 0.7 | 7 | Полный фазовый переход, защитная коррекция |

Проверяется в: `TAT-7 Comparative Benchmark` — divergence trace показывает эскалацию на пороге 0.30.

### 3.2 Принцип чайника (достаточность как энергоэффективность)

> *Вы не кипятите воду постоянно, чтобы заварить чай. Вы нагреваете её до температуры, требуемой задачей.*

7 голов — это кипяток, используемый только когда нужно. Большинство консолидаций происходит ниже точки кипения, и это корректное поведение, а не деградация.

Проверяется в: `TAT-ROOT/docs/ru/KETTLE_PRINCIPLE.md`

### 3.3 Энергоэффективность

Чанковая архитектура TAT сокращает использование памяти и токенов в **25–30 раз** по сравнению с сырыми markdown/JSON/CSV логами, сохраняя 100% значимой информации через структурный приоритет связей над признаками (`connection_weight = 4`).

Проверяется в: `TAT-ONE-TAP/docs/ru/COST_EFFICIENCY.md`
