# TAT 操作包络线

## 93–96%: 充足性区间

93–96% 范围不是质量目标。它是从 TAT 内部常数推导出的**操作包络线**。

### 下边界: 93%

93% = 100% − 7%。7 来自三个关联来源：

- **7 个头** (TAT-7 架构)
- **T_max = 0.7** (相变温度)
- **0.7% 受控噪声** (软边界)

这是下边界：系统不会低于 93%，因为此时 7 头模式与保护性校正会激活。架构会在崩溃之前捕捉到下降。

### 上边界: 96%

96% = 100% − 4%。4 来自 **connection_weight_multiplier = 4** —— 结构优先级规则：连接的重要性是特征的 4 倍。

这是上边界：系统不会追求高于 96% 的准确率，因为剩余的 4% 代表**充足性高原**，进一步改进需要指数级资源成本，而改进幅度不足 0.2%。

### 含义

93–96% 是 TAT 相变操作的区间：

- **低于 93%** → 系统升级到 7 个头
- **高于 96%** → 系统达到高原，停止消耗资源

包络线不是偏好——它是**架构的几何结构**。

## 充足性作为设计约束

充足性不是妥协。它是**设计约束**，有三个具体实现：

### 1. 自适应头部选择

TAT 使用熵驱动门控：

- **T < 0.3** → 3 个头 (冷模式，记忆稳定)
- **0.3 ≤ T < 0.7** → 5 个头 (灰色区域，保护性校正)
- **T ≥ 0.7** → 7 个头 (热模式，全相位保护)

资源充足性：只使用需要的。

### 2. 高原感知准确率目标

TAT 不优化到 100%，而是优化到操作包络线 (93–96%)。

实测超过 7 个头后每头的增益约为 0.2% —— 这不值得资源成本。架构在收益递减主导之前停止。

### 3. 标记而非阻止

7 头模式不会暂停整合。当 harmony 沉默时，它将输出标记为低密度。

架构提供信息；它不决定行动。

## 下游合同

三级输出交接：

### 级别 1: TAT 单层内部

- harmony (Depth) 将输出标记为低密度
- Position 和 Coherence 接收此信息并相应调整聚合权重

### 级别 2: TAT 层级之间

- N 层通过分形晶格将数据 + 相干性分数传递给 N+1 层
- 低密度标签随数据一起传递
- 下一层决定：直接使用、等待或请求共振重放

### 级别 3: 外部系统

- Cophy Runtime, HeartFlow 等
- 下游是任务调度器或决策模块
- 它接收 TAT 输出和相干性分数
- 如果相干性为 0.45 且 harmony 沉默，下游系统决定是否行动或延迟

**TAT 不做这个决定——它提供测量值。**

## 表面相干性 (Surface Coherence)

> 术语由 **@icophy** 提出 (DeepSeek-V3 #1447)。

**定义:** 系统活跃 (Position)、内部一致 (Coherence)，但未将新旧信息有意义地整合 (harmony 沉默) 的状态。活动无连通性。反思无整合。

**检测:** harmony 作为结构性否决。如果即使 Position 和 Coherence 很高时 harmony 仍然沉默——整合不完整，状态被标记为**表面相干性**。

**与包络线的关系:** 表面相干性可能在 93–96% 范围内发生。它不是错误状态。它是下游系统可以用来决定行动或等待的**可诊断状态**。

## 摘要

| 组件 | 描述 |
|---|---|
| **93–96%** | 从 7% 和 4% 常数导出的操作包络线 |
| **充足性** | 设计约束：自适应头部、高原感知准确率、标记而非阻止 |
| **下游** | 三级合同：内部、层级间、外部 |
| **表面相干性** | 可诊断状态：活动无连通性；术语由 @icophy 提出 |
